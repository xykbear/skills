# 贡献指南

感谢您对 prompt-creator 技能的关注！本技能旨在提供通用的系统提示词工程方法论。

## 如何贡献

### 报告问题

如果您发现：
- 核心原则（L3）有误导性内容
- OpenCode 特定机制（L2）已过时
- 示例代码有错误

请通过 PR 或 Issue 反馈。

### 添加新的 L2 工具支持

如果您使用 ClaudeCode、Cursor 或其他工具，希望扩展本技能的适用范围：

1. **创建 L2 文档**：新建 `references/L2-[ToolName].md`
2. **内容要求**：
   - 该工具的系统提示词机制（如 CLAUDE.md 结构、.cursorrules 格式）
   - 与 OpenCode/AGENTS.md 的主要差异
   - 特定语法或最佳实践
3. **更新 SKILL.md**：在 L2 章节添加对新工具的支持说明
4. **提交 PR**：简要说明该工具的用户场景

### 内容标准

所有贡献应遵循：

1. **与 L3 核心原则保持一致**：工具特定机制不得违背通用 Prompt 工程原则
2. **提供具体示例**：抽象描述配合实际代码示例
3. **标注版本**：注明该内容适用的工具版本
4. **保持简洁**：参考文档按需懒加载，不强制阅读

## 文档结构

```
prompt-creator/
├── SKILL.md              # 核心文档（通用原则）
├── CONTRIBUTING.md       # 本文件
├── references/
│   ├── 审查清单.md       # 通用审查标准
│   ├── XML标签规范.md    # 通用标签规范
│   ├── 配置架构.md       # 通用架构模式
│   ├── 输出格式.md       # 通用输出模板
│   └── L2-*.md          # 工具特定机制（按需添加）
```

## 维护说明

- **L3 核心层**：由技能维护者维护，变更需谨慎
- **L2 工具层**：由各工具用户社区维护，接受增量更新
- 工具版本更新时，L2 文档应尽量同步更新

## 许可

贡献即表示您同意将内容发布在相同的许可协议下。
