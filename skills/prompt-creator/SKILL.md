# prompt-creator

帮助用户创建、优化和沉淀系统提示词的skill。

## 概述

prompt-creator 旨在帮助用户从实际使用中提炼经验，持续优化系统提示词。类似"代码审查"但针对系统提示词工程。

## 为什么需要这个skill

系统提示词（AGENTS.md）是影响AI行为的核心配置。随着使用深入，会积累大量经验：

- **偏好**：如"简洁回复"、"不用markdown"
- **规则**：如"优先使用edit"
- **最佳实践**：如"分层级管理配置"

这些经验如果只停留在对话中，会随会话结束而丢失。这个skill帮助您将这些经验沉淀到系统提示词中，实现：
1. 避免重复解释相同偏好
2. 积累可复用的最佳实践
3. 持续优化AI行为

## 典型场景

### 场景1：提炼系统提示词
用户完成一系列会话后，想要沉淀经验

**效果较差的触发**：
- "帮我看看有什么可以优化的"
- "看看有没有遗漏"

**效果更好的触发**：
- "是否有可提炼的系统提示词"
- "把这次的经验沉淀下去"
- "我想把偏好固定下来"

**处理流程**：
1. 分析当前会话上下文，识别用户偏好、规则、限制等可沉淀内容
2. 与现有agents.md对比，检测重复或需要更新的部分
3. 生成修改建议，包含XML格式的系统提示词片段
4. 让用户确认目标层级和具体文件位置
5. 辅助执行修改
6. 提醒用户重启工具或执行热加载

### 场景2：审查现有提示词
用户请求审查或优化现有配置

**效果较差的触发**：
- "看看这个文件"

**效果更好的触发**：
- "看看当前项目的提示词是否有问题"
- "检查下配置有没有问题"
- "审查AGENTS.md"

**处理流程**：
1. 确定目标AGENTS.md文件位置（根据上下文或询问用户）
2. 对照最佳实践清单审查
3. 识别问题（格式、完整性、清晰度等）
4. 生成优化建议报告
5. 用户确认后辅助修改

## 最佳实践知识库

### 建议使用的工具

- **Read工具**：读取现有agents.md文件内容
- **Glob工具**：搜索项目中所有AGENTS.md文件
- **Grep工具**：在配置文件中搜索特定关键词
- **Edit工具**：修改确认后的内容
- **Bash工具**：如需备份（git操作）

### 核心原则

1. **明确性**：指令应清晰明确，避免歧义
2. **上下文**：提供指令背后的动机，帮助LLM理解目标
3. **示例**：用"效果更好"的示例替代"效果较差"的示例
4. **工具使用**：明确指定使用哪些工具，避免"建议vs实施"的歧义
5. **安全性**：考虑操作可逆性，要求确认风险操作
6. **格式控制**：使用XML标签引导输出格式

### 层级判断规则

#### 推荐的实践方式

AGENTS.md支持多层嵌套。这是一种推荐的实践方式，但用户可以自行决定是否使用。

**推荐的层级划分**：
- 最外层：跨项目通用配置
- 中间层：领域/工作区配置
- 最内层：项目特定配置

#### 如何确定当前层级

1. **从上下文中获取**：读取用户的AGENTS.md获取其定义的层级
2. **从路径推断**：根据当前工作目录的AGENTS.md文件位置判断
3. **询问用户**：如不明确，直接询问用户

#### 内容类型与层级对应（推荐原则）

| 内容类型 | 建议位置 |
|----------|----------|
| 跨项目通用偏好 | 最外层 |
| 特定领域规则 | 中间层 |
| 单项目特定规则 | 最内层 |
| 临时经验 | 不建议沉淀 |

#### 优先级规则
- 内层配置可以覆盖外层配置

#### 重要提醒
- **不要假设固定结构**：用户可能使用单层、双层或任意层级的AGENTS.md
- **询问用户确认**：首先确认用户的配置结构
- **层级实践参考**：读取 `references/配置架构.md` 获取推荐实践框架

## 工作流程

### 1. 会话沉淀流程

```
1. 接收用户请求
2. 分析当前会话上下文：
   - 用户的明确偏好（如"简洁回复"）
   - 重复出现的规则（如"不要用markdown"）
   - 成功的问题解决方案
   - 用户的纠正/反馈
3. 搜索现有agents.md，检测重复
4. 生成建议：
   - 提取的内容（XML格式）
   - 建议添加位置
   - 理由说明
5. 呈现给用户确认
6. 用户确认后辅助修改
7. 提醒用户重启/热加载
```

### 2. 审查优化流程

```
1. 确定目标文件（根据上下文确定层级，或询问用户）
2. 读取agents.md内容
3. 对照最佳实践清单逐项审查
4. 生成审查报告：
   - 问题列表
   - 优化建议
   - 优先级排序
5. 呈现给用户
6. 用户确认后辅助修改
7. 提醒用户重启/热加载
```

## 注意事项

1. **用户主导**：始终让用户确认修改位置和内容，不自动修改
2. **避免重复**：添加前检查现有agents.md是否已有类似内容
3. **适度沉淀**：并非所有经验都值得沉淀，避免过度工程化
4. **清晰解释**：说明为什么某内容值得沉淀及其影响
5. **风险警示**：修改 AGENTS.md 是**高风险操作**
   - 外层配置影响范围大，内层影响范围小
   - 建议修改前备份（git），修改后测试验证
6. **层级实践**：
   - 首先确认用户的配置架构（读取用户的AGENTS.md）
   - 外层定义基础规范，内层进行具体化
   - 避免在非定义层重复已存在的规则
