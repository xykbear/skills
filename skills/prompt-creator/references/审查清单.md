# 系统提示词审查清单

## 核心原则

1. **明确性**：指令应清晰明确，避免歧义
2. **上下文**：提供指令背后的动机，帮助LLM理解目标
3. **示例**：用"效果更好"的示例替代"效果较差"的示例
4. **安全性**：考虑操作可逆性，要求确认风险操作
5. **自主性**：明确默认行为倾向，确保主动执行而非仅提供建议

## 常见优化维度

| 维度 | 检查点 |
|------|--------|
| 明确性 | 指令是否清晰无歧义 |
| 上下文 | 是否提供动机/背景 |
| 安全性 | 风险操作是否需要确认 |
| 自主性 | 行为倾向是否明确 |
| 状态管理 | 是否有进度跟踪机制 |
| 边界限制 | 是否明确禁止行为 |

## 审查流程

1. 确定目标文件位置和层级
2. 逐项对照清单审查
3. 识别问题并记录
4. 生成优化建议
5. 按优先级排序输出
