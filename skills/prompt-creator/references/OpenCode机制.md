# OpenCode 系统特性（可选参考）

> 本文件是**可选参考资料**，仅在以下情况使用：
> - 用户明确使用 OpenCode 工具
> - 需要理解 OpenCode 特定机制时
> - 作为懒加载参考，不强制要求读取

---

## AGENTS.md 层级（OpenCode 特有）

| 层级 | 位置 | 加载条件 |
|------|------|----------|
| 全局 | `~/.config/opencode/AGENTS.md` | 任意目录运行 opencode 时加载 |
| 本地（当前目录） | `./AGENTS.md` | 运行目录或其父目录中存在时加载 |

**优先级**：本地配置可覆盖全局配置。

## @ 引用机制

### 解析规则

- **路径基准**：@ 引用相对于**运行目录**解析，而非 AGENTS.md 所在目录
- **框架行为**：框架自动注入，非模型主动读取
- **失败表现**：引用失败时模型不会收到错误提示，可能基于不完整信息回答

### 验证方法

观察日志：
- `→ Read <file>` - 文件被框架加载
- `permission=read` - 权限检查过程
- 无相关日志 - 引用未生效

### 常见误区

| 误区 | 真相 |
|------|------|
| @ 引用失败会导致报错 | 框架沉默失败，模型可能不知道 |
| 子目录运行时引用自动失效 | 取决于运行目录是否有权限访问引用路径 |
| 全局配置可以引用外部文件 | 受权限限制，建议内容内联 |

## 分层设计意图

### 规则层 vs 实现层

| 层级 | 特征 | 示例 |
|------|------|------|
| 规则层 | 定义"不准"、"必须"的约束 | 全局 AGENTS.md 中的禁止规则 |
| 实现层 | 解释"how"的实现细节 | 引用文件中的具体操作说明 |

**判断原则**：两者内容相似不一定是重复，可能是分层设计的体现。

---

## WHEN 触发条件

### 作用

`when:` 条件触发是 OpenCode 推荐的实践：
- 模型在意图匹配时自动加载对应引用
- 不需要手动索引或目录

### 判断原则

有 WHEN 触发条件 ≠ 需要索引，触发条件本身就是索引的替代方案。

---

## 验证优先原则

在提出问题前，必须先验证：

1. **使用调试命令**
   ```bash
   opencode debug config   # 查看配置解析
   opencode debug skill   # 查看技能加载
   ```

2. **观察实际行为**
   ```bash
   opencode run "test" --print-logs --log-level DEBUG
   ```

3. **确认问题真实存在**
   - 表面现象 ≠ 实际问题
   - 设计决策 ≠ 问题

---

## 不是问题的"表面问题"

| 表面现象 | 实际判断 | 处理方式 |
|----------|----------|----------|
| 多处风险提示 | 可能是静态+动态分层保护 | 验证是否各有其上下文意义 |
| 规则层与实现层内容相似 | 可能是分层设计 | 理解"不准"vs"how"的区别 |
| 术语未解释 | 模型能理解即可 | 不强制要求解释 |
| 缺少索引/目录 | 可能有 WHEN 触发条件 | 验证触发条件是否有效 |
