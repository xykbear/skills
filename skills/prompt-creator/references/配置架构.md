# AGENTS.md 配置架构参考

## 说明

这是一种推荐的实践方式，而非必须遵循的规范。用户可以根据自己的需求选择使用单层、双层或多层架构。

## 常见模式（推荐实践）

### 模式A：三层架构

适用于需要区分全局/工作区/项目的用户。

| 层级 | 典型位置 | 典型职责 |
|------|----------|----------|
| 外层 | 用户主目录下的AGENTS.md | 跨项目通用规范 |
| 中层 | 工作区根目录的AGENTS.md | 领域/工作区规范 |
| 内层 | 项目目录的AGENTS.md | 项目特定规范 |

### 模式B：两层架构

适用于只需要全局+项目的用户。

| 层级 | 典型位置 | 典型职责 |
|------|----------|----------|
| 外层 | 用户主目录下的AGENTS.md | 通用规范 |
| 内层 | 项目目录的AGENTS.md | 项目特定规范 |

### 模式C：单层架构

仅有项目级配置，无统一全局规范。

## 确定用户架构的步骤

1. 读取用户的AGENTS.md（通常在用户主目录或工作区根目录）
2. 观察其中定义的层级数量
3. 如不明显，直接询问用户

## 优先级规则

- 内层配置可以覆盖外层配置
- 这允许在特定项目中灵活定制

## 风险级别（供参考）

| 配置层级 | 风险 | 说明 |
|----------|------|------|
| 外层 | 较高 | 影响范围大 |
| 中层 | 中等 | 影响部分项目 |
| 内层 | 较低 | 仅影响特定项目 |

### 风险示例

**外层配置错误示例**：
```
在外层AGENTS.md中配置：
"默认删除所有未跟踪文件"

影响：所有使用此全局配置的项目都可能意外删除文件
```

**中层配置冲突示例**：
```
工作区配置："使用2空格缩进"
项目配置："使用4空格缩进"

问题：开发者可能困惑于不同项目中的不同表现
```

**修改前建议**：
- 外层修改：先在单个项目测试，确认无问题后再应用到外层
- 中层修改：检查该工作区下所有项目的影响范围
- 内层修改：风险较低，但仍建议备份或版本控制

## 执行建议

技能执行时应：
1. 从当前上下文推断配置结构
2. 或直接询问用户确认
3. 不要假设固定的层级数量
