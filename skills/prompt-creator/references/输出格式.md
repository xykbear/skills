# 输出格式模板

## 结构化输出示例

使用 XML 标签组织审查报告：

```xml
<review_report>
  <file_name>AGENTS.md</file_name>
  <issues>
    <issue severity="high" priority="1">
      <description>XML标签命名不一致</description>
      <suggestion>统一使用snake_case</suggestion>
    </issue>
    <issue severity="medium" priority="2">
      <description>缺少上下文说明</description>
      <suggestion>添加"为什么"的解释</suggestion>
    </issue>
  </issues>
  <warnings>
    <warning>修改外层配置影响范围大，建议先测试</warning>
  </warnings>
</review_report>
```

## 何时使用哪种格式

**会话沉淀建议输出**：
- 使用场景：用户完成会话后，想要沉淀经验到系统提示词
- 触发词："沉淀一下"、"把经验固定下来"、"修改系统提示词"

**审查报告输出**：
- 使用场景：用户请求审查现有配置
- 触发词："检查配置"、"审查AGENTS.md"、"看看有没有问题"

## 会话沉淀建议输出

```markdown
## 可添加到系统提示词的内容

### 1. [建议名称]
**来源**：从本次会话中识别

**内容**：
```xml
<tag_name>
具体内容
</tag_name>
```

**建议层级**：[根据上下文确定]
**理由**：为什么建议添加

---
添加到哪个位置？
- [ ] 外层配置（跨项目通用）
- [ ] 中间层配置（领域/工作区）
- [ ] 内层配置（项目特定）
- [ ] 其他（由用户指定）
```

## 审查报告输出

```markdown
## 审查报告：[文件名]

### 问题列表

| # | 问题 | 严重程度 | 建议 |
|---|------|----------|------|
| 1 | [问题描述] | 高/中/低 | [建议修改] |

### 优化建议

1. [建议1]
2. [建议2]

### 风险提示
[如涉及高风险修改，给出警示]
```
